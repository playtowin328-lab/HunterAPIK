# APK Converter Android Agent

Это первый Android-агент для связки с Telegram Mini App.

Что он делает:

- хранит `Server URL`, `Owner ID`, `DEVICE_API_TOKEN` и имя устройства
- определяет Android-модель устройства
- запускает foreground service с постоянным уведомлением
- отправляет heartbeat на `/api/devices/heartbeat` каждые 30 секунд
- после перезагрузки снова стартует, если был включен

## Сборка

1. Открой папку `android_agent` в Android Studio.
2. Дождись Gradle sync.
3. Подключи Android-телефон по USB.
4. Нажми Run.

Если Gradle попросит скачать Android Gradle Plugin или SDK 35, разреши установку.

Если APK не устанавливается:

- удали старую версию `APK Agent` с телефона и установи заново
- проверь, что Android версии 8.0+
- если ставишь APK файлом, включи установку из неизвестных источников для файлового менеджера/браузера
- лучше ставить через Android Studio кнопкой `Run`, так APK будет debug-signed автоматически
- если ошибка `App not installed as package conflicts`, значит на телефоне уже стоит APK с тем же package id, но другой подписью

## Настройка

В основном проекте запусти бота:

```bash
python main.py
```

В боте отправь:

```text
/pair
```

Самый быстрый способ:

1. Открой ссылку из сообщения `/pair`.
2. Android предложит открыть `APK Agent`.
3. Агент сам заполнит `Server URL` и код.
4. После успешного Pair нажми `Старт агента`.

Ручной способ:

- `Server URL` — адрес компьютера с ботом, например `http://192.168.1.10:8080`
- `Pairing code` — код из `/pair`
- `Device name` — например `POCO C75`

Ещё можно скопировать `apkagent://...` ссылку или 6-значный код и нажать `Вставить ссылку/код` в Android Agent.

Важно: с телефона `http://127.0.0.1:8080` означает сам телефон, а не компьютер.
Нужен LAN IP компьютера, например `192.168.1.10`.

Если используешь WireGuard, укажи VPN-адрес backend:

```text
http://10.66.66.1:8080
```

WireGuard-шаблоны лежат в `wireguard/`.

После этого нажми:

1. `Сохранить`
2. `Pair по коду`
3. `Тест heartbeat`
4. `Старт агента`
5. `Открыть настройки жестов` и включи `APK Agent`, если нужно управлять тапами

Устройство должно появиться в мини-аппе как `Online`.

`Owner ID` и `Token` оставлены как расширенный ручной режим. В обычном сценарии они не нужны: после `Pair по коду` приложение само сохранит привязку и персональный секрет устройства.

Кнопка `Экран` в мини-аппе отправляет запрос агенту. Android покажет системное окно разрешения записи экрана; после подтверждения агент начнёт отправлять JPEG-кадры на backend примерно раз в 2 секунды.

После появления preview можно нажимать прямо по картинке экрана в мини-аппе. Агент выполнит тап через Accessibility Service, если он включён в настройках Android.
Также доступны кнопки `Back`, `Home`, `Recent` и `Стоп экран`.
Добавлены свайпы `Swipe ↑`, `Swipe ↓`, `Swipe ←`, `Swipe →`.
Поле `Текст на телефон` вставляет текст в активное поле ввода на Android. Для этого тоже нужен включённый Accessibility Service.

Агент также отправляет в мини-апп статус телефона: батарею, зарядку, тип сети, версию Android, модель и включён ли Accessibility Service.

Кнопки `Файлы` и `Команды` пока забираются агентом из очереди и подтверждаются в статусе, но сами модули файлов/сложных жестов ещё не включены.

## Что это не делает пока

Агент пока не управляет экраном, не читает файлы и не выполняет команды.
Сейчас он только подтверждает, что устройство подключено и живое.
