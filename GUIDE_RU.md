# Hunter Agent: инструкция по установке и использованию

Hunter Agent состоит из трех частей:

- Telegram-бот: запускает сборку APK, выдает QR и показывает диагностику.
- Android Agent: приложение на твоем телефоне.
- Telegram Mini App: пульт устройств, экран, жесты и команды.

Проект рассчитан на личные устройства. Просмотр экрана, жесты и фоновая работа включаются только после подтверждения на телефоне.

## 1. Проверка Railway

В боте нажми `Полная проверка` или отправь:

```text
/check
```

Должно быть:

- `PUBLIC_BASE_URL` — HTTPS-адрес Railway.
- `MINI_APP_URL` — HTTPS-адрес мини-апа.
- `GITHUB_TOKEN` — нужен для сборки APK.
- `APK download` — `OK`, если APK уже собран.

## 2. Сборка APK

### Как выбрать режим

| Режим | Когда выбирать | Что умеет |
| --- | --- | --- |
| Lite APK | Если нужно просто подключить телефон, видеть Online/Offline, батарею, сеть и проверить связь | QR-подключение, heartbeat, базовый статус |
| Full APK | Если нужно видеть экран и управлять своим Android из мини-апа | Экран, tap, long tap, swipe, Back/Home/Recent, ввод текста, уведомления, быстрые настройки |

В боте открой `Подключить телефон` и нажми `Lite или Full: что выбрать?`.
Для простой установки нажимай `Собрать APK`. Для управления экраном нажимай `Собрать Full APK для экрана и управления` или используй команду `/build_apk_full Hunter Agent Full`.

Быстрый вариант:

```text
/build_apk
```

Сборка со своим названием:

```text
/build_apk Hunter Agent
```

Сборка со своим значком:

1. Отправь картинку боту.
2. Отправь `/build_apk Hunter Agent`.
3. Дождись сообщения со ссылкой на APK.

По умолчанию публикуется Lite APK. Он подключает телефон к боту и мини-апу, но не содержит Accessibility, запись экрана, автозапуск и запрос отключения батарейной оптимизации. Это сделано, чтобы Google Play Защита реже блокировала установку.

Полная сборка с экраном и жестами запускается отдельно:

```text
/build_apk_full Hunter Agent Full
```

Full APK может получать предупреждения Play Protect, потому что функции Accessibility и записи экрана относятся к чувствительным remote-control возможностям.

APK публикуется в GitHub Release:

```text
https://github.com/playtowin328-lab/HunterAPIK/releases/download/android-agent-latest/apk-agent.apk
```

## 3. Установка на Android

1. Открой на телефоне страницу:

```text
https://web-production-715d7.up.railway.app/agent
```

2. Нажми `Скачать APK`.
3. Если Android блокирует установку, нажми `Настройки`.
4. Разреши установку из текущего браузера или файлового менеджера.
5. Вернись к APK и подтверди установку.

Android может показывать предупреждение, потому что APK установлен не из Play Market. Это нормальная системная защита. Ее нельзя и не нужно обходить скрыто: установку подтверждает владелец телефона. Если Play Protect полностью блокирует Full APK, используй Lite APK или публикуй приложение через Google Play internal testing.

Легальные варианты для Full APK:

- установить только на свое устройство и подтвердить предупреждения вручную;
- временно отключить Play Protect на своем телефоне в настройках Google Play, если ты понимаешь риск;
- использовать Google Play internal testing/закрытый трек;
- использовать MDM/Android Enterprise для своих корпоративных устройств;
- ставить через ADB на устройстве с включенной отладкой.

## 4. Подключение телефона

В боте нажми `Подключить телефон` или отправь:

```text
/pair
```

Дальше:

1. Открой QR-ссылку на телефоне.
2. Нажми `Открыть Android Agent`.
3. Приложение само подставит сервер и код.
4. После успешного подключения агент запустит мастер разрешений.

## 5. Разрешения Android Agent

Агент попросит только те разрешения, которые Android разрешает запросить явно:

- Уведомления — нужны для foreground service.
- Lite APK: уведомления и сеть для связи с ботом.
- Full APK: работа в фоне, Accessibility и запись экрана для расширенного управления.

Accessibility и запись экрана нельзя включить автоматически из приложения. Android специально требует ручного подтверждения владельцем устройства.

## 6. Работа в мини-апе

Открой мини-ап из бота:

- `Скачать APK` — страница установки агента.
- `Получить QR` — бот пришлет QR в чат.
- `Обновить` — обновить список устройств.
- `Экран` — запросить системное разрешение записи экрана на телефоне.
- `Back`, `Home`, `Recent`, `Swipe` — работают после включения Accessibility.

Если телефон не появился:

1. Открой Android Agent.
2. Нажми `Проверить связь`.
3. Проверь, что сервер начинается с `https://web-production-715d7.up.railway.app`.
4. В боте нажми `Мои устройства`.

## 7. Частые проблемы

`APK not ready`  
Запусти `/build_apk` и дождись успешной сборки GitHub Actions.

`Приложение не устанавливается`  
Удали старую версию с тем же package id и установи заново. Если Android блокирует источник, разреши установку из браузера/файлового менеджера.

`Телефон offline`  
Открой Android Agent, нажми `Запустить агент` и `Проверить связь`. Проверь разрешение фоновой работы.

`Экран не показывается`  
В мини-апе нажми `Экран`, затем на телефоне подтверди системное окно записи экрана.

`Жесты не работают`  
На телефоне включи Accessibility service `Hunter Agent`.

## 8. PC Agent для ПК и VDS

Собрать Windows EXE:

```text
/build_pc_agent
```

Скачать последнюю сборку:

```text
https://github.com/playtowin328-lab/HunterAPIK/releases/download/pc-agent-latest/hunter-pc-agent.exe
```

Подключить ПК:

```powershell
hunter-pc-agent.exe pair --server https://web-production-715d7.up.railway.app --code 123456 --name "Home PC"
hunter-pc-agent.exe run
```

ADB-мост для Android рядом с ПК:

```powershell
hunter-pc-agent.exe setup --server https://web-production-715d7.up.railway.app --code 123456 --name "Home PC" --startup
```

Это самый простой режим: одна команда привязывает PC Agent, проверяет ADB, включает мост и добавляет автозапуск Windows.

Как подготовить телефон, который останется дома:

1. Установи Android Platform Tools на ПК и добавь `adb` в PATH.
2. На телефоне включи `Для разработчиков` и `USB debugging` или `Wireless debugging`.
3. Подключи телефон к ПК и подтверди RSA-ключ на экране телефона.
4. Запусти команду `setup` выше. В мини-апе появится отдельное устройство `adb-...`.
5. Кнопка `Экран`, тапы, свайпы, Back/Home/Recents и ввод текста будут идти через официальный ADB.

ADB работает только с твоим физически/локально авторизованным устройством. Без подтверждения debugging на самом телефоне мост не подключится.

Если нужно проверить, что все готово:

```powershell
hunter-pc-agent.exe doctor --adb
```

Если автозапуск не нужен, можно запускать вручную:

```powershell
hunter-pc-agent.exe pair --server https://web-production-715d7.up.railway.app --code 123456 --name "Home PC"
hunter-pc-agent.exe run --adb --interval 3
```

Для поездок схема такая: домашний ПК должен быть включен, PC Agent должен быть в автозапуске, а телефон должен быть подключен по USB или заранее настроенному Wireless debugging. Тогда ты открываешь мини-ап из любой страны и работаешь через домашний мост.

Для полноценного экрана ПК используй нормальный канал:

- Windows/VDS: WireGuard + RDP.
- Linux VDS: WireGuard + SSH.
- Кроссплатформенно: RustDesk или Chrome Remote Desktop.

PC Agent не скрывается, не ставит автозапуск и не выполняет произвольные команды. Это база для легального подключения своих машин.
