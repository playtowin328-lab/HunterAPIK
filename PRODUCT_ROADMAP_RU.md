# Hunter Platform: правильная архитектура

## 1. Device Control Layer

Цель: стабильный доступ к своим устройствам.

- Android Agent: автономный APK для телефона, когда нет домашнего ПК.
- ADB Bridge: быстрый режим через домашний ПК, USB/Wireless debugging и PC Agent.
- PC Agent: домашний мост, автозапуск, диагностика, heartbeat.
- Mini App: единый пульт, экран, жесты, статусы, быстрые команды.

Главное правило: управление устройством всегда видимое и только после явного разрешения владельца.

## 2. Trading Layer

Для реальной крипто-торговли основной путь должен быть через официальные API бирж, а не через клики по экрану.

Минимальные блоки:

- Exchange connectors: Binance, Bybit, OKX и другие API.
- Paper trading: тестовый режим без реальных денег.
- Risk engine: лимиты на сделку, дневной лимит убытка, max leverage, stop loss.
- Strategy engine: стратегии отдельно от исполнения.
- Execution engine: размещение ордеров, отмена, retry, проверка статуса.
- Audit log: журнал каждого сигнала, решения и ордера.
- Secrets vault: API keys только в защищенном хранилище, не в коде и не в чате.
- Alerting: Telegram-уведомления о сделках, ошибках и лимитах.

Телефонный remote-control здесь нужен как fallback: посмотреть приложение биржи, подтвердить действие вручную или проверить состояние, но не как основной торговый робот.

## 3. Что улучшаем ближайшими шагами

1. Устойчивый live screen для ADB и Android Agent.
2. Очередь команд с приоритетами: жесты важнее кадров экрана.
3. Диагностика задержки: ping, command latency, frame age.
4. Профили устройств: дом, VDS, телефон, резервный телефон.
5. Paper trading модуль без API-ключей.
6. После тестов — безопасное подключение API биржи.

## 5. Уже заложено

- ADB live screen сам запрашивает свежие кадры.
- Очередь команд отдает жесты и кнопки раньше кадров.
- Мини-ап показывает frame age, очередь, последнюю команду и задержку подтверждения.
- PC Agent отдает loop time, screen capture time, command time и последнюю ошибку.
- Мини-ап умеет переключать live screen профили: быстро, баланс, качество.

## 4. Почему так

Если делать торговлю через экран телефона, задержки, блокировки, всплывающие окна и лаги могут привести к ошибкам на деньгах. API-подход дает контроль, журналирование и риск-лимиты. Remote-control оставляем как человеческий аварийный пульт.
